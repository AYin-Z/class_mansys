# 区队管理系统 - 产品需求文档 (PRD)

## 项目概述

### 项目名称

区队管理系统（微信小程序）

### 开发单位

中国人民公安大学数据警务技术六区区队综合事务管理系统开发组

### 版本信息

- 当前版本：v0.0.2 (Dev)
- 开发阶段：功能完善期

### 项目定位

本系统是一款面向中国人民公安大学数据警务技术六区的综合性事务管理平台，旨在简化班级日常事务处理流程，提高班级管理效率，为区队成员提供便捷的事务管理与信息交流平台。

## 核心功能架构

### 一、用户身份管理系统

#### 1.1 用户注册功能

**功能描述：**

- 强制注册机制：新用户首次进入小程序时强制展示注册页面
- 信息采集：
  - 基本信息：学号、真实姓名、手机号码、邮箱地址、性别
  - 身份信息：班级选择、政治面貌（群众/共青团员/中共预备党员/中共党员）
  - 职务信息：班级职务（无/区队长/生活副区/学习副区/心理副区/团支书/组织委员/宣传委员）、学校组织职务
- 安全验证：管理员身份需输入超级密码验证（密码：kskblzdjdwqzkbl）

**技术实现：**

- 注册信息加密存储（Base64模拟加密）
- 与微信openid唯一绑定，防止重复注册
- 注册成功后自动跳转至首页并完成自动登录
- 用户信息实时同步至"我的"页面显示

#### 1.2 角色权限体系

**管理员角色定义（7个）：**

| 角色   | 职责       | 特殊权限         |
| ---- | -------- | ------------ |
| 区队长  | 全面负责区队工作 | 最高审批权、班费最终审批 |
| 生活副区 | 生活事务管理   | 资金保管员、班费收缴   |
| 学习副区 | 学习事务管理   | 作业发布权限       |
| 心理副区 | 心理健康关注   | 心理干预申请处理     |
| 团支书  | 团务工作管理   | 团员活动组织       |
| 组织委员 | 组织建设管理   | 记账员、班费记账     |
| 宣传委员 | 宣传工作管理   | 公告发布         |

**权限控制机制：**

- 基于角色的访问控制（RBAC）
- 页面级权限控制（如审批页面仅管理员可操作）
- 功能级权限控制（如发布作业仅学习副区和课代表）
- 动态权限检测，实时响应角色变化

### 6.3 班费使用申请功能

**申请分类（5类）：**

- 文体活动
- 学风建设
- 公共物品购置
- 同学慰问
- 其他集体性支出

**分级审批流程：**

**小额申请（<100元）：**

1. 经办人提交申请 → 系统自动审核通过

**中额申请（100-500元）- 四步标准化流程：**

1. 经办人提交详细申请（用途说明、金额明细、证明材料）
2. 区队长初步审核（合理性与合规性）
3. 班干部集体投票（需≥2/3同意）
4. 投票通过后执行使用并按规定报销

**大额申请（>500元）：**

1. 经办人提交预算方案
2. 区队长和指导员双重审核
3. 全班匿名投票（需19票以上同意）
4. 执行使用并报销

**审批轨迹记录：**

- 各环节处理人记录
- 处理时间戳
- 审批意见记录
- 完整审计链路

## 已知限制与后续规划

### 当前限制

1. **数据存储**：使用本地存储，无后端服务支持
2. **支付功能**：个人小程序限制，无法实现在线支付
3. **文件上传**：模拟实现，实际部署需对接云存储
4. **消息推送**：依赖微信订阅消息能力
5. **并发处理**：单机运行，无服务器支持

### 后续优化方向

1. **后端服务接入**：实现真正的数据持久化和多用户协作
2. **支付系统集成**：对接微信支付或校园卡系统
3. **云存储服务**：文件上传至云端
4. **消息推送优化**：实时通知提醒
5. **性能优化**：大数据量下的性能提升
6. **安全加固**：更完善的加密和认证机制
7. **用户体验**：界面美化和交互优化
