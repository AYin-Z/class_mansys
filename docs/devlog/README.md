## 写作约定

每篇日志建议至少包含以下章节：

1. **范围 / 主题**：本次迭代涉及的模块与目标
2. **改动详情**：按"前端 / 后端 / 设计令牌 / 工具层"等维度分组
3. **修复的关键问题**：表格列出 #问题 → 修复
4. **待办（backlog）**：留给下一轮的改进项
5. **受影响文件清单**：新增 / 修改两类
