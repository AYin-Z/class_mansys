## Phase 0 — 添加辅导员角色

### 0.1 角色常量 + 权限矩阵

**涉及文件**: `src/constants/roles.ts`

```diff
+ COUNSELOR: 9,
```

- 加到 `USER_ROLES` 常量
- 加到 `ROLE_LABELS` → `'辅导员'`
- 加到 `ADMIN_ROLE_IDS`（辅导员也算管理角色）
- 审批相关权限：`APPROVE_FEE_USE` 加入 `COUNSELOR`，`APPROVE_LEAVE` 加入 `COUNSELOR`

### 0.2 后端校验同步

**涉及文件**: `backend/models/ExpenseApproval.js`

- `determineSteps()`: step 2 的 `role` 从 `null` 改为 `9`（辅导员）
- `getPendingApprovals()`: `OR (ea.step = 2)` → `OR (ea.step = 2 AND ? = 9)` 增加角色过滤
- `approveByAdvisor()`: 已存在，确认逻辑正确

**涉及文件**: `backend/controllers/FeeController.js`

- `approveExpense()`: 增加 `isCounselor` 校验 `role === 9`
- step 2 仅允许 role=9 审批

## Phase 1 — P0 修复

### 1.1 审批投票链路修复

**涉及文件**: `src/pages/fee/apply-approve.vue`

当前 `apply-approve.vue` 只有「通过/驳回」按钮，大额审核进入 step 3（投票）后无人可操作。

**改动**:
- step=3 投票态的申请，展示投票进度条（当前票数 / 19 票阈值）
- 所有班干部可见投票入口 + 可投「同意/反对」
- 前端调 `POST /api/fee/approvals/:id/vote`

**涉及文件**: `backend/controllers/FeeController.js`

- `castVote` 增加角色校验：仅限班干部投票（`role >= 1 && role <= 8`）
- 投票阈值 19 保持（38 人班，≥2/3）

### 1.2 辅导员审批走通

依赖于 Phase 0（角色定义）。完成后链路完整：
- ≤100: 承办人 → 区队长 → 执行 ✅
- 100-500: 承办人 → 区队长 → **辅导员** → 执行（修复后生效）
- >500: 承办人 → 区队长 → **辅导员复核** → 投票 → 执行

## Phase 2 — P1 修复

### 2.1 收缴操作前端入口

**涉及文件**: `src/pages/fee/collection.vue`

当前只展示收缴状态列表，无操作按钮。需要添加：
- 管理员可见「发起收缴」按钮 → 弹窗填写金额/标题 → `POST /collections`
- 已缴纳成员不可再操作，未缴纳成员可见「缴纳」按钮 → `POST /collections/:id/pay`
- 管理员可见「标记免缴」按钮（人均）
- 管理员可见「截止收缴」按钮

### 2.2 金额校验（前后端）

**前端** (`src/pages/fee/apply.vue` + `reimbursement.vue`):
- `parseFloat` 后检查 `isNaN`、`<= 0`、`> 100000`（上限）
- 显示即时错误提示

**后端** (`FeeController.js`):
- `createExpense`: 校验 `amount > 0 && amount <= 100000`
- 失败返回 400

### 2.3 证明材料上传

**涉及文件**: `src/pages/fee/apply.vue`

- `chooseImage` 选图后，调用 `uni.uploadFile` 上传到后端
- 后端接收文件存到 `uploads/` 目录
- `createExpense` 调用时传入 `proof_url`
